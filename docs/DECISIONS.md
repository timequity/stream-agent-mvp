# Decisions

## 2026-05-22

Decision: clarify that the format uses setup-aware verdicts, while multi-setup comparisons are optional.

Reason: the show should avoid claiming that "AI" succeeded or failed universally when the result depends on the specific model, context, tools, files, workflow, and human control.

What this prevents: rigid episode structure, false AI-vs-human conclusions, shallow model comparisons, and overgeneralized verdicts.

Review date: after Stream 001.

## 2026-05-21

Decision: adapt Stream 001 from a generic OBS/AI assistant setup stream into Episode 0 / Live Lab of **"Справится ли AI? / Can AI Handle This?"**

Reason: the new format gives the stream a clear question, criteria, scorecard, and public hook.

What this prevents: turning the stream into an unfocused dev session, AI-tool overview, or premature production build.

Review date: after Stream 001.
